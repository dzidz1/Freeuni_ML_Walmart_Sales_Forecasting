# Walmart Recruiting - Store Sales Forecasting

## Kaggle კონკურსის მოკლე მიმოხილვა

პროექტის მიზანი იყო Walmart-ის 45 მაღაზიის ყოველკვირეული გაყიდვების პროგნოზირება. კონკრეტული მაღაზია შეიცავს რამდენიმე დეპარტამენტს და ჩვენ გვევალებოდა სეილების პროგნოზი თითოეული მაღაზიის თითოეული დეპარტამენტისთვის (Store x Department ცალი კომბინაცია). სულ მონაცემებში 3331 ასეთი კომბინაცია, ანუ 3331 დამოუკიდებელი დროითი მწკრივი გვაქვს. პროექტზე მუშაობისას გამოვცადეთ დროით მწკრივების პროგნოზირების რამდენიმე არქიტექტურული ოჯახი:

- **Classical Statistical Models** — ARIMA, SARIMA, Prophet
- **Tree-Based Models** — LightGBM, XGBoost
- **Deep Learning** — N-BEATS, Temporal Fusion Transformer, PatchTST, DLinear
- **Foundation Model** — TimesFM

ყველა არქიტექტურა შეფასდა ერთსა და იმავე ვალიდაციის სქემაზე და ერთი და იმავე მეტრიკით, რომელიც განსაზღვრული იყო კონკურსისთვის, კერძოდ, ეს გახლდათ WMAE. მაშასადამე, მათი შედეგები პირდაპირ შედარებადია და ამ დეტალებს და შეფასების მეთოდოლოგიას README-ში დაწვრილებით გაეცნობით.

## მონაცემები

გადმოგვეცემოდა შემდეგი ფაილები: train.csv მთავარი სატრენინგო ფაილი სვეტებით Store, Dept, Date, IsHoliday და Weekly_Sales (სამიზნე). მონაცემები მოიცავს 02/2012-10/2020 პერიოდს. შემდეგი ფაილია test.csv, რომელსაც აქვს იგივე სტრუქტურა, თუმცა Weekly_Sales-ის გარეშე და პროგნოზი კეთდება 11/2012-07/2013 პერიოდზე. ასევე გვაქვს features.csv ფაილი დამატებითი ცვლადებისთვის, როგორებიცაა: ტემპერატურა, საწვავის ფასი, Markdown1-5 (ფასდაკლებები), CPI და უმუშევრობის დონე. ბოლო ფაილი კი გახლავთ stores.csv, საიდანაც ვიგებთ ინფორმაციას მაღაზიის ტიპისა და ზომის შესახებ. დამატებით ცნობილია 4 მნიშვნელოვანი დღის თარიღები: Super Bowl, Labor Day, მადლიერების დღე და შობა, რომელთათვისაც სადღესასწაულო კვირების შეცდომა 5-მაგად ისჯება.

## შეფასების მეთოდოლოგია

იმისთვის, რომ სხვადასხვა არქიტექტურის შედეგების შედარებას აზრი ჰქონდეს, ყველა ექსპერიმენტში ერთი და იგივე ვალიდაციის სქემა გამოვიყენეთ. თითოეული არქიტერქტურის მოდელისთვის data-wise split ასეთია: სატრენინგო კვირის ბოლო 13 კვირა გამოყოფილია ვალიდაციისთვის და მოდელი მხოლოდ მანამდელ მონაცემებზე სწავლობს. უნდა ვახსენოთ, რომ დროითი მწკრივებისთვის random split არასწორ სურათს იძლევა (მოდელი ვერ უნდა ხედავდეს მომავალს), რაც პროგნოზირების რეალურ ამოცანას არ შეესაბამება.
